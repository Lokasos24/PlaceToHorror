## Funciones con especificadores
- Funcion con **const**: Es para decirle a la funcion que los datos son solo de lectura, es decir, las variables que se pasan no pueden modificarse.

- Funcion con **static**: Le dice a la funcion que solo le pertenece a esa *clase*, para llamarla con otra funcion se le debe declarar *static* si no esta generaliza la funcion con todas las *entidades* creadas por esa clase.

- Funcion con **virtual**: La funcion permite ser sobreescrita por una *clase* hija, esto es para que busque quien contiene la *verdad* de la funcion, es como un intermediario que dirije donde va.

- Funcion con **inline**: Esta es para funciones mas pequeñas, para que se pegue el codigo directamente y no tenga que buscarse.

- Funcion con **explicit**: Evitas que C++ haga converciones que no quieras, necesitas ser claro con lo que redactaste.
